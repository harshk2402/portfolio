Event‑Driven CAAR Analysis: Earnings Surprise Study

C++
Event Study
Multithreading
Repository

An end‑to‑end, multithreaded earnings‑surprise event study in C++. The system classifies equities into Beat/Meet/Miss portfolios, bootstraps resamples for stability, and computes abnormal return (AR), average abnormal return (AAR), and cumulative AAR (CAAR) over an event window. Data are pulled programmatically (libcurl) and results are plotted with gnuplot for quick diagnostics.

Data & Universe

  • Price & benchmark: Daily closes for constituents; benchmark index (e.g., IWV) for market‑adjusted returns
  • Events: Earnings announcement dates and surprises (actual vs. consensus)
  • Windows: Pre‑event, event day (t=0), and post‑event horizons (e.g., −20 … +20)

Methodology

  • Grouping: Map each ticker to Beat, Meet, or Miss based on surprise thresholds
  • Bootstrapping: Resample K names × N trials per group to reduce idiosyncratic noise
  • Metrics: Compute AR, AAR (cross‑sectional average), and CAAR (cumulative AAR) for each group
  • Comparisons: Contrast CAAR across Beat/Meet/Miss; inspect AAR around t=0

System Architecture

  • Retrieval: libcurl fetch for prices/EPS; robust parsing & date alignment
  • Engine: Multithreaded portfolio construction and bootstrap trials; thread‑safe aggregation
  • Outputs: CSV summaries (per‑group AAR/CAAR), gnuplot scripts & rendered charts

Validation & Controls

  • Input sanity checks (symbol coverage, date ranges, missing data)
  • Event alignment guardrails (trading‑day mapping around t=0)
  • Bootstrap convergence checks (trials vs. variance)

Results (Brief)

  • Beat group tends to show positive CAAR post‑announcement
  • Miss group underperforms around and after t=0
  • Meet group close to flat; effect sizes vary by sample

Limitations & Next Steps

  • Refine surprise thresholds and sector/size controls
  • Add heteroskedasticity‑robust inference and multiple‑testing adjustments
  • Parallelize plotting and I/O; add CLI flags for windows and bootstrap params

Key Features

  • Multithreaded retrieval and portfolio computation
  • Beat / Meet / Miss classification with bootstrapped AAR/CAAR
  • Clean plots via gnuplot; CSV outputs for further analysis
